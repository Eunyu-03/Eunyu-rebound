## 📝 7. 느낀 점

### 📌 기획: 충분한 사전 기획이 프로젝트의 완성도를 좌우한다

이번 프로젝트는 기획 단계가 짧았던 탓에 작업 중간에 방향을 재조정하는 일이 많았다.  
처음 시도하는 유형의 프로젝트였던 만큼, 무엇을 구상해야 하는지 명확하지 않은 상태에서 시작하게 되었고,  
결과적으로 구현 과정에서 팀원들과 논의하며 결정해야 할 부분이 많았다.  
이 경험을 통해 사전 기획의 중요성을 절실히 느꼈으며, 다음 프로젝트에서는 기능 정의와 사용자 흐름, 기술 스택 선정 등을 보다 체계적으로 준비할 계획이다.

---

### 🤝 협업: 작은 이슈라도 빠른 공유가 핵심

총 4명이 함께 진행한 협업 프로젝트였기에 커뮤니케이션의 중요성을 크게 체감했다.  
예를 들어, pull request 전 pull을 받지 않아 다른 팀원의 작업물이 덮어지는 상황이 발생했고,  
이를 통해 작업 전후의 절차를 철저히 지키는 습관이 필요함을 배웠다.  
또한 시스템 오류나 예기치 못한 트러블이 생겼을 때, 문제를 빠르게 공유하고 함께 해결해 나가는 과정이  
프로젝트의 안정성을 높이는 데 큰 도움이 되었다.

---

### 🚀 시행착오 속에서도 성장한 경험

기획부터 구현까지 뜻대로 되지 않는 부분이 많아 아쉬움도 있었지만,  
막연했던 아이디어가 기술을 통해 실제로 구현되는 과정을 지켜보며 큰 보람을 느꼈다.  
다양한 기술적 요소를 직접 다뤄보며 감각을 키울 수 있었다.  
이번 프로젝트는 시행착오를 통해 성장할 수 있었던 값진 경험이었으며,  
다음 프로젝트에서는 이 배움을 바탕으로 더욱 완성도 높은 결과물을 만들어내고 싶다.
